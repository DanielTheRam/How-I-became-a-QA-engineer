My Journey how I eventually became a QA tester

This project serves as a personal summary of what I studied before getting my first job in IT. It's not a portfolio or a showcase of HTML/CSS skills — just a way to track and share my learning path. Hopefully, it can serve as motivation for others starting out in testing.


Where it all begun

Khan Academy

I started by learning the basics of HTML and CSS through Khan Academy. Understanding these basics has helped me even as a tester, especially when analyzing websites or working with DevTools.
Check out the materials: Khan Academy


NaučmeIT

This Czech platform helped me go deeper into QA topics. It also recommended Khan Academy at the beginning.
Both sites offer free access to content about HTML, CSS, DevTools, and more.
Check it out here: NaučmeIT
Author's LinkedIn: BleedingDev | GitHub: GitHub


Radek Kitner – 3-Month Course: "How to Become a Tester"

I joined a 3-month QA course led by Radek Kitner. The course was very practical, not just theory.

Topics I Learned:

  • Creating test cases
  • Writing in TestRail
  • Bug reporting in Jira
  • Basic SQL commands
  • SCRUM methodology
  • Types of testing
  • API testing in Postman
  • Analyzing network traffic using Wireshark
  • Using browser DevTools

GEN Academy (Avast) – ISTQB Prep Course

I had the opportunity to attend an ISTQB CTFL prep course, which I completed with the final exam.

Topics I Learned:

  • Fundamentals of Testing
  • Testing Throughout the SDLC
  • Static Testing
  • Test Techniques
  • Test Management
  • Tool Support
  • Practical experience with black-box, static, and exploratory testing

Python Basics

During the ISTQB course, I also learned Python with Jaroslav Kafka. He introduced basic data types and explained Git in more depth.
Although Python wasn’t ideal for web testing, it helped me understand programming fundamentals.
Today, I mostly use JavaScript and TypeScript, but the experience with Python made code easier to read and understand.

Test Automation – Online Course

I took an automation course that covered:

  • Theoretical background of automation
  • Basic use of PyCharm
  • Basics of Robot Framework

Unfortunately, the course was a bit outdated and didn’t add much value.


My First Job in IT

In January 2024, I landed my first IT job as a QA Engineer (software tester).
The beginning was tough, but it helped me understand how things really work in a company.
Cheers to all junior testers — stay consistent, work on your own projects, explore, and don’t stop applying!
Most importantly: don't push too hard. The goal is to enjoy what you do.
Good luck everybody! Daniel #2025
